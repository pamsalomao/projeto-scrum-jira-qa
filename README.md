#  Projeto Scrum no Jira – Fluxo de Trabalho QA

#  Objetivo

Configurar um projeto utilizando a metodologia ágil Scrum no Jira, simulando um fluxo real de desenvolvimento com foco em testes de software.



#  Metodologia Utilizada

- Scrum
- Sprint com duração de 2 semanas
- Organização por Backlog e Board



#  Estrutura do Projeto

Foram criados os seguintes tipos de issues:

- Epic
- Story
- Task
- Bug

As atividades foram organizadas no Backlog e posteriormente adicionadas à Sprint.



#  Fluxo de Trabalho Definido

To Do → Em Desenvolvimento → Em Teste → Aguardando Correção → Concluído



#  Descrição das Etapas

#  To Do
Itens planejados para a Sprint e ainda não iniciados.

#  Em Desenvolvimento
Funcionalidade está sendo desenvolvida pelo time.

#  Em Teste
Funcionalidade aguardando validação do QA.
Erros foram identificados: enviado a Aguardando Correção.
Erros corrigidos pelo desenvolvimento: Bugs foram corrigidos e validados por QA. (enviado a Concluído)

#  Aguardando Correção
Bugs identificados: tarefa retorna para o desenvolvimento.

#  Concluído
Funcionalidade desenvolvida, testada e validada.



# Funcionamento do Processo

1. A issue é criada no Backlog.
2. Durante o planejamento, é adicionada à Sprint.
3. Inicia em To Do.
4. Vai para Em Desenvolvimento.
5. Após conclusão do desenvolvimento, segue para Em Teste.
6. Se aprovado, vai para Concluído.
7. Se reprovado, vai para Aguardando Correção e retorna ao desenvolvimento.
